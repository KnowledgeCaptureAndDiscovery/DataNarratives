Data narratives for result: term-topic matrix visualization

Data Narrative 1: Execution view (see more details)

The topic detection method was run on the Reuters_R8_train and Englisch_Stopwords datasets, with iterations set to 100, number of topics set to 10 and batch size set to 10. The term-topic matrix visualization results are stored online (DOI https://dx.doi.org/10.6084/m9.figshare.3406603.v1).

Data Narrative 2: Data view (see more details)

The term-topic matrix visualization results have been derived from the Reuters_R8_train and Englisch_Stopwords datasets.

Data Narrative 3: Method view (see more details)

TopicDetection performs one main type of analysis on the analyzed datasets. The TrainTopics step produces the main results of the workflow, after two filtering steps (StopWords and Smallwords) and a reformatting step (FormatDataset).

The term-topic matrix visualization is the result of the PlotTopics component, a visualization step.

Data Narrative 4: Implementation view (see more details)

TopicDetection performs one main type of analysis on the analyzed datasets. The TrainTopics step produces the main results of the workflow, after two filtering steps (StopWords and Smallwords) and a reformatting step (FormatDataset). TrainTopics has been implemented with the Parallel LDA software.

The term-topic matrix visualization is the result of the PlotTopics component, a visualization step. The PlotTopics component has been implemented with the Termite software.

Data Narrative 5: Step view (see more details)

The TopicDetection method has five steps. The first one, Stop words step, uses an input dataset and a words dataset to produce a filtered result. Next, the Small words step consumes that output to produce another filtered result. The next step is the Format dataset a reformatting step which adapts the result for the Train topics step. Next, the TrainTopics step produces an output topics dataset. Finally the Plot topics step is takes the output topics dataset to create the term-topic matrix visualization.

Data Narrative 6: Software view (see more details)

The TopicDetection method uses two main software components for its functionality. The first one, Parallel LDA, is used to implement the TrainTopics and produce the output topics dataset. Parallel LDA is a parallel C++ implementation of fast Gibbs sampling of Latent Dirichlet Allocation with Apache License 2.0. The second software is Termite, which implements the Plot topics step to create the term-topic matrix visualization. Termite (Termite: Visualization Techniques for Assessing Textual Topic Models) has a Copyright (c) 2013, Leland Stanford Junior University.
